Expectations

The Expectations module implements methods to verify one or more values against set of expectations. This is a subset of design-by-contract programming (see en.wikipedia.org/wiki/Design_by_contract) features, and should speed you up during development phases.

Example

This function expects a String argument starting with "http:", an Integer or Float argument, and a Hash with a String entry at key :foo, and either an Array or nil at key :bar. 

  def function(a, b, options = {})
    expect! a => /^http:/, 
            b => [Integer, Float], 
            options => {
              :foo => String,
              :bar => [ Array, nil ]
            }
  end

expect(*expectations, &block) click to toggle source

Verifies a number of expectations. If one or more expectations are not met it raises an ArgumentError.

This method can be enabled or disabled at runtime using Expectations.enable and Expectations.disable.
